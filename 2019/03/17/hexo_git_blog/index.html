借助Hexo&GitHub搭建个人博客

2019年03月17日 |
技术/ Hexo
| Hexo GitHub

许多学生与在职人员希望能维护一个聚焦于内容的个人博客，但迫于服务器昂贵的价格、冗长繁琐的开发周期以及高昂的维护成本，一直没有得以实现。

Hexo&GitHub很好的解决了这个问题：Hexo是一款基于Node.js的静态博客框架;GitHub Pages为用户提供支持部署静态页面的平台。两者结合，可快速搭建出美观实用的静态网页。

本文中将介绍Hexo安装、Hexo静态网页生成、GitHub Pages部署。

Hexo官网 : https://hexo.io/zh-cn/
GitHub Pages官网 : https://pages.github.com/

Hexo

Hexo是一个快速、简介且高效的博客框架，基于Node.js，使用Markdown解析文章，并可在极端时间内渲染出美观的静态网页。

GitHub Pages部署

在开始之前，你首先得确保完成两件事情:

  1. 注册GitHub账号
  2. 电脑本地上安装好Git

GitHub建立个人仓库

步骤如下:

  1. 登陆GitHub
  2. 找到并点击按钮”New reponsitory”
  3. 创建一个新的、公共的reponsitory,并且命名为<username>.github.io,此处的<username>代表的是你注册GitHub的用户名。
  4. 完成创建

完成创建之后，GitHub会自动将这个特殊的代码仓库绑定至GitHub Pages上，同时提供访问的域名:https://<username>.github.io/。

当我们访问这个域名的时候，GitHub会默认地指向仓库中的index.html文件，并以静态页面展示给用户。

为了能在本地无密上传GitHub,你需要在GitHub上配置在本地上生成的SSH公钥，之后便可更快捷、无密地上传文件，而不用次次输入用户名和密码来登陆。

系统将会在用户目录下的.ssh文件夹中生成两个文件:id_rsa、id_rsa.pub,分别为ssh私钥与公钥。
Windows 文件夹:C:\Users\Administrator.ssh
Linuex 目录:~/.ssh/

  1. 打开id_rsa.pub，复制其内容
  2. 打开GitHub网站，点击右上角用户头像,点击setting,点击左侧导航栏”SSH and GPG keys”按钮，点击”New SSH Key”按钮
  3. 为允许无密登录你GitHub的电脑起个名字，输入Title,并将拷贝下来的公钥内容复制在Key中，点击添加按钮。

至此，SSH配置已经成功，注意不要删除与修改.ssh文件夹中两个密钥的任何东西。
